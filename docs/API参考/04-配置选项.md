# 配置选项参考

## 配置方式

项目支持三种配置方式，优先级从高到低：

1. **代码配置** - 运行时动态设置
2. **环境变量** - 系统环境变量
3. **配置文件** - TOML格式配置文件

---

## 配置文件

### 文件位置

配置文件按以下顺序查找（优先级从高到低）：

1. `configs/mijiaAPI.toml` - 项目根目录的configs目录（推荐）
2. `config.toml` - 项目根目录
3. `~/.mijia/config.toml` - 用户主目录
4. SDK自带的默认配置 - 最低优先级

### 配置文件格式

```toml
# config.toml

[api]
base_url = "https://api.mina.mi.com"

[network]
default_timeout = 30
max_retries = 3
retry_delay = 1

[cache]
device_list_ttl = 300        # 设备列表：5分钟
device_state_ttl = 30         # 设备状态：30秒
device_spec_ttl = 31536000    # 设备规格：1年（365天）
home_list_ttl = 300           # 家庭列表：5分钟

[redis]
enabled = false
host = "localhost"
port = 6379
db = 0
password = ""
key_prefix = "mijia:"

[logging]
level = "INFO"
format = "json"
file = "logs/mijia.log"
max_bytes = 10485760
backup_count = 5

[credential]
default_path = "~/.mijia/credential.json"
auto_refresh = true
```

## API配置

### base_url

API基础URL。

- **类型**：`str`
- **默认值**：`"https://api.mina.mi.com"`
- **环境变量**：`MIJIA_API_BASE_URL`
- **说明**：米家API的基础地址

**示例**：
```toml
[api]
base_url = "https://api.mina.mi.com"
```

```bash
# 环境变量
export MIJIA_API_BASE_URL="https://api.mina.mi.com"
```

```python
# 代码配置
config = ConfigManager()
config.set("API_BASE_URL", "https://api.mina.mi.com")
```

## 网络配置

### default_timeout

默认请求超时时间（秒）。

- **类型**：`int`
- **默认值**：`30`
- **环境变量**：`MIJIA_NETWORK_DEFAULT_TIMEOUT`
- **说明**：HTTP请求的默认超时时间

**示例**：
```toml
[network]
default_timeout = 30
```

### max_retries

最大重试次数。

- **类型**：`int`
- **默认值**：`3`
- **环境变量**：`MIJIA_NETWORK_MAX_RETRIES`
- **说明**：网络请求失败时的最大重试次数

**示例**：
```toml
[network]
max_retries = 3
```

### retry_delay

重试延迟（秒）。

- **类型**：`int`
- **默认值**：`1`
- **环境变量**：`MIJIA_NETWORK_RETRY_DELAY`
- **说明**：重试之间的基础延迟时间（使用指数退避）

**示例**：
```toml
[network]
retry_delay = 1
```

**重试策略**：
- 第1次重试：等待 1秒
- 第2次重试：等待 2秒
- 第3次重试：等待 4秒

## 缓存配置

### device_list_ttl

设备列表缓存时间（秒）。

- **类型**：`int`
- **默认值**：`300` (5分钟)
- **环境变量**：`MIJIA_CACHE_DEVICE_LIST_TTL`
- **说明**：设备列表的缓存有效期

**示例**：
```toml
[cache]
device_list_ttl = 300
```

### device_state_ttl

设备状态缓存时间（秒）。

- **类型**：`int`
- **默认值**：`30`
- **环境变量**：`MIJIA_CACHE_DEVICE_STATE_TTL`
- **说明**：设备状态的缓存有效期

**示例**：
```toml
[cache]
device_state_ttl = 30
```

### device_spec_ttl

设备规格缓存时间（秒）。

- **类型**：`int`
- **默认值**：`31536000` (1年)
- **环境变量**：`MIJIA_CACHE_DEVICE_SPEC_TTL`
- **说明**：设备规格的缓存有效期，使用很长的 TTL 实现长期缓存

**示例**：
```toml
[cache]
device_spec_ttl = 31536000  # 1年（365天）
```

### home_list_ttl

家庭列表缓存时间（秒）。

- **类型**：`int`
- **默认值**：`300` (5分钟)
- **环境变量**：`MIJIA_CACHE_HOME_LIST_TTL`
- **说明**：家庭列表的缓存有效期

**示例**：
```toml
[cache]
home_list_ttl = 300
```

## Redis配置

### enabled

是否启用Redis缓存。

- **类型**：`bool`
- **默认值**：`false`
- **环境变量**：`MIJIA_REDIS_ENABLED`
- **说明**：启用后使用Redis作为分布式缓存

**示例**：
```toml
[redis]
enabled = true
```

```bash
export MIJIA_REDIS_ENABLED=true
```

### host

Redis服务器地址。

- **类型**：`str`
- **默认值**：`"localhost"`
- **环境变量**：`MIJIA_REDIS_HOST`

**示例**：
```toml
[redis]
host = "redis.example.com"
```

### port

Redis服务器端口。

- **类型**：`int`
- **默认值**：`6379`
- **环境变量**：`MIJIA_REDIS_PORT`

**示例**：
```toml
[redis]
port = 6379
```

### db

Redis数据库编号。

- **类型**：`int`
- **默认值**：`0`
- **环境变量**：`MIJIA_REDIS_DB`

**示例**：
```toml
[redis]
db = 0
```

### password

Redis密码。

- **类型**：`str`
- **默认值**：`""`
- **环境变量**：`MIJIA_REDIS_PASSWORD`

**示例**：
```toml
[redis]
password = "your_password"
```

```bash
export MIJIA_REDIS_PASSWORD="your_password"
```

### key_prefix

Redis键前缀。

- **类型**：`str`
- **默认值**：`"mijia:"`
- **环境变量**：`MIJIA_REDIS_KEY_PREFIX`
- **说明**：所有Redis键的前缀，用于命名空间隔离

**示例**：
```toml
[redis]
key_prefix = "mijia:prod:"
```

## 日志配置

### level

日志级别。

- **类型**：`str`
- **默认值**：`"INFO"`
- **环境变量**：`MIJIA_LOG_LEVEL`
- **可选值**：`DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`

**示例**：
```toml
[logging]
level = "DEBUG"
```

```bash
export MIJIA_LOG_LEVEL=DEBUG
```

### format

日志格式。

- **类型**：`str`
- **默认值**：`"json"`
- **环境变量**：`MIJIA_LOG_FORMAT`
- **可选值**：`json`, `text`

**示例**：
```toml
[logging]
format = "json"
```

**JSON格式输出**：
```json
{
  "timestamp": "2026-03-13T12:00:00Z",
  "level": "INFO",
  "message": "获取设备列表",
  "user_id": "123456",
  "home_id": "789",
  "duration_ms": 150
}
```

**文本格式输出**：
```
2026-03-13 12:00:00 INFO 获取设备列表 user_id=123456 home_id=789 duration_ms=150
```

### file

日志文件路径。

- **类型**：`str`
- **默认值**：`"logs/mijia.log"`
- **环境变量**：`MIJIA_LOG_FILE`

**示例**：
```toml
[logging]
file = "/var/log/mijia/app.log"
```

### max_bytes

日志文件最大大小（字节）。

- **类型**：`int`
- **默认值**：`10485760` (10MB)
- **环境变量**：`MIJIA_LOG_MAX_BYTES`
- **说明**：超过此大小后自动轮转

**示例**：
```toml
[logging]
max_bytes = 10485760  # 10MB
```

### backup_count

日志文件备份数量。

- **类型**：`int`
- **默认值**：`5`
- **环境变量**：`MIJIA_LOG_BACKUP_COUNT`
- **说明**：保留的历史日志文件数量

**示例**：
```toml
[logging]
backup_count = 5
```

## 配置管理器

### ConfigManager

配置管理器类，用于读取和管理配置。

```python
from mijiaAPI_V2.core.config import ConfigManager

# 创建配置管理器
config = ConfigManager()

# 或指定配置文件
config = ConfigManager(config_path=Path("my_config.toml"))
```

### 读取配置

```python
# 获取配置值
base_url = config.get("API_BASE_URL")
timeout = config.get("NETWORK_DEFAULT_TIMEOUT")

# 获取配置值（带默认值）
max_retries = config.get("NETWORK_MAX_RETRIES", default=3)

# 获取布尔值
redis_enabled = config.get_bool("REDIS_ENABLED")

# 获取整数值
redis_port = config.get_int("REDIS_PORT")
```

### 设置配置

```python
# 设置配置值
config.set("API_BASE_URL", "https://api.example.com")
config.set("NETWORK_DEFAULT_TIMEOUT", 60)
config.set("REDIS_ENABLED", True)
```

### 配置优先级

```python
# 1. 代码设置（最高优先级）
config.set("REDIS_ENABLED", True)

# 2. 环境变量
# export MIJIA_REDIS_ENABLED=false

# 3. 配置文件（最低优先级）
# [redis]
# enabled = false

# 实际值：True（代码设置优先）
```

## 使用示例

### 基本配置

```python
from pathlib import Path
from mijiaAPI_V2 import create_api_client
from mijiaAPI_V2.core.config import ConfigManager

# 创建配置
config = ConfigManager(config_path=Path("config.toml"))

# 修改配置
config.set("NETWORK_DEFAULT_TIMEOUT", 60)
config.set("CACHE_DEVICE_LIST_TTL", 600)

# 创建API客户端
api = create_api_client(credential)
```

### 生产环境配置

```toml
# production.toml

[api]
base_url = "https://api.mina.mi.com"

[network]
default_timeout = 60
max_retries = 5
retry_delay = 2

[cache]
device_list_ttl = 600
device_state_ttl = 60

[redis]
enabled = true
host = "redis.production.com"
port = 6379
password = "${REDIS_PASSWORD}"
key_prefix = "mijia:prod:"

[logging]
level = "WARNING"
format = "json"
file = "/var/log/mijia/app.log"
max_bytes = 52428800  # 50MB
backup_count = 10
```

```toml
# development.toml

[api]

[network]
```


## 配置验证

```python
from mijiaAPI_V2.core.config import ConfigManager

def validate_config(config: ConfigManager) -> bool:
    """验证配置"""
    # 检查必需配置
    if not config.get("API_BASE_URL"):
        print("错误：缺少API_BASE_URL配置")
        return False
    
    # 检查Redis配置
    if config.get_bool("REDIS_ENABLED"):
        if not config.get("REDIS_HOST"):
            print("错误：启用Redis但未配置REDIS_HOST")
            return False
    
    # 检查超时配置
    timeout = config.get_int("NETWORK_DEFAULT_TIMEOUT")
    if timeout < 1 or timeout > 300:
        print(f"警告：超时时间 {timeout} 不在推荐范围(1-300)")
    
    return True

# 使用
config = ConfigManager()
if validate_config(config):
    print("配置验证通过")
else:
    print("配置验证失败")
```

## 最佳实践

### 1. 使用环境变量管理敏感信息

```toml
# config.toml
[redis]
password = "${REDIS_PASSWORD}"  # 从环境变量读取
```

```bash
export REDIS_PASSWORD="your_secret_password"
```

### 2. 不同环境使用不同配置

```python
import os
from pathlib import Path

# 根据环境选择配置文件
env = os.getenv("ENV", "development")
config_file = Path(f"configs/{env}.toml")

config = ConfigManager(config_path=config_file)
```

### 3. 配置文件版本控制

```gitignore
# .gitignore
config.toml          # 不提交本地配置
production.toml      # 不提交生产配置
*.local.toml         # 不提交本地配置

# 提交模板配置
config.example.toml  # 提交配置模板
```

### 4. 配置文档化

```toml
# config.example.toml
# 这是配置文件模板，复制为 config.toml 并修改

[api]
# API基础URL，通常不需要修改
base_url = "https://api.mina.mi.com"

[network]
# 请求超时时间（秒），建议30-60
default_timeout = 30

# 最大重试次数，建议3-5
max_retries = 3
```
