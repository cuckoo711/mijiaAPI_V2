## 登录问题

### 二维码登录成功

**当前状态**：二维码登录功能已修复并正常工作。

**使用方法**：
```python
from mijiaAPI_V2 import create_auth_service

# 创建认证服务
auth_service = create_auth_service()

# 二维码登录
credential = auth_service.login_by_qrcode()

# 保存凭据
auth_service.save_credential(credential)
```

**登录流程**：
1. 运行登录代码后会显示二维码
2. 使用米家APP扫描二维码
3. 在APP中确认登录
4. 等待约2分钟（长轮询超时时间）
5. 登录成功后凭据会自动保存

**注意事项**：
- 二维码有效期约2分钟，超时需要重新生成
- 确保网络连接正常
- 使用最新版本的米家APP扫码

### 二维码登录失败（历史问题，已修复）

**症状**（已修复）：
- 运行二维码登录时提示"登录验证失败"
- 错误代码：70016
- 错误描述：登录验证失败

**解决方案**：
SDK已更新登录实现，使用与旧版本相同的登录API：
- 使用 `longPolling/loginUrl` 端点获取二维码
- 使用长轮询（120秒超时）等待扫码
- 从callback URL获取cookies和token

如果仍然遇到登录问题，请尝试：

**诊断方法**：
```bash
# 运行登录诊断脚本（可选）
uv run python scripts/diagnose_login.py
```

**如果登录仍然失败**：

1. **检查网络连接**
   ```bash
   # 测试是否能访问米家服务器
   curl https://account.xiaomi.com
   ```

2. **使用已有凭据**
   如果之前已经成功登录过，可以使用保存的凭据：
   ```python
   from mijiaAPI_V2 import create_api_client_from_file
   
   # 从文件加载凭据（默认路径：~/.mijia/credential.json）
   api = create_api_client_from_file()
   ```

3. **检查米家APP版本**
   - 确保使用最新版本的米家APP
   - 旧版本APP可能无法正常扫码

4. **查看详细日志**
   ```python
   import logging
   logging.basicConfig(level=logging.DEBUG)
   
   # 然后运行登录代码
   ```

**手动创建凭据（高级用法）**：

如果你已经有 `serviceToken` 和 `ssecurity`（例如从浏览器中提取），可以手动创建凭据：

```python
from datetime import datetime, timedelta
from mijiaAPI_V2.domain.models import Credential
from mijiaAPI_V2 import create_api_client

# 手动创建凭据
credential = Credential(
    user_id="your_user_id",
    service_token="your_service_token",
    ssecurity="your_ssecurity",
    device_id="your_device_id",
    user_agent="iOS-14.4-6.0.103-iPhone12,1",
    expires_at=datetime.now() + timedelta(days=7)
)

# 创建API客户端
api = create_api_client(credential)

# 保存凭据以便后续使用
from mijiaAPI_V2 import create_auth_service
auth_service = create_auth_service()
auth_service.save_credential(credential)
```

## 凭据过期问题

### 凭据过期无法刷新

**症状**：
- 调用API时提示 `TokenExpiredError`
- 刷新凭据失败

**解决方案**：

1. **检查凭据状态**
   ```python
   from mijiaAPI_V2 import create_auth_service
   
   auth_service = create_auth_service()
   credential = auth_service.load_credential()
   
   if credential:
       print(f"是否过期: {credential.is_expired()}")
       print(f"剩余时间: {credential.expires_in()} 秒")
   ```

2. **尝试刷新凭据**
   ```python
   try:
       new_credential = auth_service.refresh_credential(credential)
       auth_service.save_credential(new_credential)
       print("凭据刷新成功")
   except Exception as e:
       print(f"凭据刷新失败: {e}")
       print("需要重新登录")
   ```

3. **重新登录**
   如果刷新失败，需要重新登录获取新凭据。

**症状**：

**解决方案**：

1. **检查网络连接**
   ```bash

   proxies = {
       "http://": "http://proxy.example.com:8080",
       "https://": "http://proxy.example.com:8080",
   }
   ```

## 缓存问题

### 缓存数据不一致

**症状**：
- 获取的设备状态与实际不符
- 控制设备后状态未更新

**解决方案**：

1. **主动刷新缓存**
   ```python
   # 刷新特定家庭的缓存
   api.refresh_cache(home_id="123456")
   
   # 刷新所有缓存
   api.refresh_cache()
   ```

2. **清空所有缓存**
   ```python
   # 谨慎使用！会清空所有用户的缓存
   api.clear_all_cache()
   ```

3. **禁用缓存**
   修改配置文件，将缓存TTL设置为0（不推荐）：
   ```toml
   [cache]
   device_list_ttl = 0
   device_state_ttl = 0
   ```

## 设备控制问题

### 设备控制失败

**症状**：
- `control_device()` 返回 `False`
- 抛出 `DeviceNotFoundError` 或 `PropertyReadOnlyError`

**解决方案**：

1. **检查设备是否在线**
   ```python
   device = api.get_device(device_id)
   if device and device.is_online():
       print("设备在线")
   else:
       print("设备离线")
   ```

2. **验证属性参数**
   - 确认 `siid`、`piid` 和 `value` 参数正确
   - 查看设备规格文档确认属性是否可写
   - 检查属性值范围是否合法

3. **查看详细错误信息**
   ```python
   try:
       api.control_device(device_id, siid, piid, value)
   except Exception as e:
       print(f"控制失败: {e}")
       import traceback
       traceback.print_exc()
   ```

## 性能问题

### API调用缓慢

**症状**：
- API调用响应时间长
- 批量操作耗时过长

**解决方案**：

1. **使用缓存**
   确保缓存功能已启用，避免重复请求。

2. **使用批量操作**
   ```python
   # 不好：逐个控制
   for device_id in device_ids:
       api.control_device(device_id, 2, 1, True)
   
   # 好：批量控制
   requests = [
       {"device_id": did, "siid": 2, "piid": 1, "value": True}
       for did in device_ids
   ]
   api.batch_control_devices(requests)
   ```

3. **使用异步API**
   ```python
   import asyncio
   from mijiaAPI_V2 import create_async_api_client
   
   async def main():
       api = create_async_api_client(credential)
       
       # 并发获取多个家庭的设备
       results = await asyncio.gather(
           api.get_devices(home_id_1),
           api.get_devices(home_id_2),
           api.get_devices(home_id_3),
       )
   
   asyncio.run(main())
   ```

4. **使用Redis缓存**
   对于多进程或分布式部署，使用Redis共享缓存：
   ```python
   import redis
   from mijiaAPI_V2 import create_api_client
   
   redis_client = redis.Redis(host='localhost', port=6379, db=0)
   api = create_api_client(credential, redis_client=redis_client)
   ```

## 401未授权错误（已修复）

### 问题描述
API请求返回401错误或JSON解析失败，表示认证或加密实现有问题。

### 根本原因
新项目的加密实现与米家API要求不一致，导致请求失败。主要问题包括：

1. **API Base URL不正确**
   - 错误：`https://api.io.mi.com/app`
   - 正确：`https://api.mijia.tech/app`

2. **API端点路径不正确**
   - 错误：`/homeroom/gethome`
   - 正确：`/v2/homeroom/gethome_merged`

3. **加密算法实现不完整**
   - RC4加密需要先丢弃前1024字节的密钥流
   - 签名算法使用SHA1而不是HMAC-SHA256
   - nonce生成格式需要符合米家API要求
   - 缺少响应解密功能

4. **HTTP请求头缺少必需字段**
   - 缺少 `Content-Type: application/x-www-form-urlencoded`
   - 缺少 `miot-accept-encoding: GZIP`
   - 缺少 `miot-encrypt-algorithm: ENCRYPT-RC4`

5. **响应处理不正确**
   - 响应内容是加密的，需要使用RC4解密
   - 如果响应头包含 `Miot-Content-Encoding: GZIP`，解密后还需要GZIP解压

### 修复内容

SDK已完全修复此问题，参考旧项目 `mijiaAPI/apis.py` 和 `mijiaAPI/miutils.py` 的实现：

1. **更新API配置**
   ```toml
   # configs/mijiaAPI.toml
   [api]
   base_url = "https://api.mijia.tech/app"
   ```

2. **修复加密实现**
   - RC4加密前丢弃1024字节密钥流
   - 使用SHA256生成signed_nonce
   - 使用SHA1生成请求签名
   - 正确的nonce生成格式（8字节随机数 + 时间戳分钟数）

3. **完整的HTTP请求头**
   ```python
   headers = {
       "User-Agent": credential.user_agent,
       "Content-Type": "application/x-www-form-urlencoded",
       "accept-encoding": "identity",
       "miot-accept-encoding": "GZIP",
       "miot-encrypt-algorithm": "ENCRYPT-RC4",
       "x-xiaomi-protocal-flag-cli": "PROTOCAL-HTTP2",
       "Cookie": "...",
   }
   ```

4. **实现响应解密**
   ```python
   # 解密响应
   signed_nonce = get_signed_nonce(ssecurity, nonce)
   decrypted = rc4_decrypt(response.text, signed_nonce)
   
   # 如果是GZIP压缩的，需要解压
   try:
       result = decrypted.decode('utf-8')
   except UnicodeDecodeError:
       result = gzip.decompress(decrypted).decode('utf-8')
   ```

5. **修复设备列表API**
   - 端点：`/home/home_device_list`（而非 `/home/device_list`）
   - 需要 `home_owner` 参数（家庭所有者的UID）
   - 支持分页获取（使用 `start_did` 和 `has_more`）
