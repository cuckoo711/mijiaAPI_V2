## 📝 文档维护规则

### 文档分类

1. **功能文档**：功能说明、使用指南（持续维护）
2. **技术文档**：架构设计、技术实现、部署指南
3. **开发文档**：代码规范、开发指南、测试说明
4. **归档文档**：历史记录、完成报告（归档到 `归档/` 目录）

### 文档规范

- 文档文件名使用中文（例如：`XXX指南.md`）
- 文档内容使用简体中文
- 代码示例必须可运行
- 包含完整的参数说明和示例

### 文档更新

- 创建、修改或删除文档时必须同步更新本索引
- 临时性文档完成后移动到 `归档/` 目录
- 在索引中标记已归档文档
