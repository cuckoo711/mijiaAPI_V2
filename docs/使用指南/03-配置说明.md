# 配置说明

本文档详细说明如何配置米家API SDK以适应不同的使用场景。

## 配置方式概览

项目支持三种配置方式，按优先级从高到低：

1. **代码配置** - 运行时动态设置（最高优先级）
2. **环境变量** - 系统环境变量
3. **配置文件** - TOML格式配置文件（最低优先级）

---

## 快速开始

### 配置文件准备

项目提供了配置文件模板，首次使用需要创建实际配置文件：

```bash
# 复制模板文件（推荐）
cp configs/default.toml.template configs/mijiaAPI.toml

# 根据需要修改配置
vim configs/mijiaAPI.toml
```

**说明**：
- `default.toml.template` 是配置模板，会提交到版本控制
- `mijiaAPI.toml` 是实际使用的配置文件，已在 `.gitignore` 中忽略
- 这样可以安全地修改配置而不用担心提交敏感信息

**配置文件查找顺序**（优先级从高到低）：
1. `configs/mijiaAPI.toml` - 项目根目录的configs目录（推荐）
2. `config.toml` - 项目根目录
3. `~/.mijia/config.toml` - 用户主目录
4. SDK自带的默认配置 - 最低优先级

### 使用默认配置

最简单的方式是使用默认配置：

```python
from mijiaAPI_V2 import create_api_client

# 使用默认配置
api = create_api_client(credential)
```

默认配置适用于大多数场景，包括：
- API基础URL
- 30秒请求超时
- 3次重试
- 5分钟设备列表缓存
- 不使用Redis

### 使用配置文件

创建 `config.toml` 文件：

```toml
[network]
default_timeout = 60
max_retries = 5

[cache]
device_list_ttl = 600

[logging]
level = "DEBUG"
```

使用配置文件：

```python
from pathlib import Path
from mijiaAPI_V2 import create_api_client

api = create_api_client(
    credential=credential,
    config_path=Path("config.toml")
)
```

### 使用环境变量

```bash
# 设置环境变量
export MIJIA_NETWORK_DEFAULT_TIMEOUT=60
export MIJIA_CACHE_DEVICE_LIST_TTL=600
export MIJIA_LOG_LEVEL=DEBUG

# 运行程序
python your_script.py
```

## 常见配置场景

### 场景1：开发环境

**需求**：
- 详细的调试日志
- 短缓存时间（便于测试）
- 不使用Redis

**配置文件**：

```toml
# configs/development.toml

[network]
default_timeout = 30
max_retries = 3

[cache]
device_list_ttl = 60      # 1分钟缓存
device_state_ttl = 10     # 10秒缓存

[redis]
enabled = false

[logging]
level = "DEBUG"
format = "text"           # 文本格式更易读
file = "logs/dev.log"
```

**使用**：

```python
from pathlib import Path
from mijiaAPI_V2 import create_api_client

api = create_api_client(
    credential=credential,
    config_path=Path("configs/development.toml")
)
```

### 场景2：生产环境

**需求**：
- 较长的超时时间
- 更多重试次数
- 使用Redis分布式缓存
- JSON格式日志

**配置文件**：

```toml
# configs/production.toml

[network]
default_timeout = 60
max_retries = 5
retry_delay = 2

[cache]
device_list_ttl = 600     # 10分钟缓存
device_state_ttl = 60     # 1分钟缓存

[redis]
enabled = true
host = "redis.production.com"
port = 6379
password = "${REDIS_PASSWORD}"
key_prefix = "mijia:prod:"

[logging]
level = "WARNING"
format = "json"
file = "/var/log/mijia/app.log"
max_bytes = 52428800      # 50MB
backup_count = 10
```

**环境变量**：

```bash
export REDIS_PASSWORD="your_secure_password"
```

**使用**：

```python
from pathlib import Path
from mijiaAPI_V2 import create_api_client
import redis
import os

# 创建Redis客户端
redis_client = redis.Redis(
    host='redis.production.com',
    port=6379,
    password=os.getenv('REDIS_PASSWORD'),
    decode_responses=True
)

# 创建API客户端
api = create_api_client(
    credential=credential,
    config_path=Path("configs/production.toml"),
    redis_client=redis_client
)
```

### 场景3：测试环境

**需求**：
- 快速失败（短超时）
- 不重试（便于测试错误处理）
- 不使用缓存

**配置文件**：

```toml
# configs/testing.toml

[network]
default_timeout = 5       # 5秒超时
max_retries = 0           # 不重试

[cache]
device_list_ttl = 0       # 不缓存
device_state_ttl = 0

[redis]
enabled = false

[logging]
level = "DEBUG"
format = "text"
file = "logs/test.log"
```

### 场景4：高并发场景

**需求**：
- 使用异步API
- Redis分布式缓存
- 较长的缓存时间
- 连接池优化

**配置文件**：

```toml
# configs/high-concurrency.toml

[network]
default_timeout = 30
max_retries = 3

[cache]
device_list_ttl = 900     # 15分钟缓存
device_state_ttl = 120    # 2分钟缓存

[redis]
enabled = true
host = "redis.cluster.com"
port = 6379
key_prefix = "mijia:hc:"

[logging]
level = "INFO"
format = "json"
```

**使用**：

```python
import asyncio
from pathlib import Path
from mijiaAPI_V2 import create_async_api_client

async def main():
    api = create_async_api_client(
        credential=credential,
        config_path=Path("configs/high-concurrency.toml"),
        redis_client=redis_client
    )
    
    # 并发获取多个家庭的设备
    homes = await api.get_homes()
    devices_list = await asyncio.gather(*[
        api.get_devices(home.id) for home in homes
    ])

asyncio.run(main())
```

## 网络配置详解

### 超时配置

```toml
[network]
default_timeout = 30
```

**选择建议**：
- **快速网络**：10-20秒
- **一般网络**：30秒（默认）
- **慢速网络**：60秒
- **不稳定网络**：90-120秒

**代码配置**：

```python
from mijiaAPI_V2.core.config import ConfigManager

config = ConfigManager()
config.set("NETWORK_DEFAULT_TIMEOUT", 60)
```

### 重试配置

```toml
[network]
max_retries = 3
retry_delay = 1
```

**重试策略**：
- 使用指数退避算法
- 第1次重试：等待 1秒
- 第2次重试：等待 2秒
- 第3次重试：等待 4秒

**选择建议**：
- **稳定网络**：max_retries = 2
- **一般网络**：max_retries = 3（默认）
- **不稳定网络**：max_retries = 5

**禁用重试**：

```toml
[network]
max_retries = 0
```

## 缓存配置详解

### 缓存时间配置

```toml
[cache]
device_list_ttl = 300        # 设备列表：5分钟
device_state_ttl = 30        # 设备状态：30秒
device_spec_ttl = 31536000   # 设备规格：1年（365天）
home_list_ttl = 300          # 家庭列表：5分钟
```

**选择建议**：

| 数据类型 | 变化频率 | 推荐TTL | 说明 |
|---------|---------|---------|------|
| 设备列表 | 低 | 300-600秒 | 设备很少增删 |
| 设备状态 | 高 | 10-60秒 | 状态经常变化 |
| 设备规格 | 极低 | 31536000秒（1年） | 规格基本不变 |
| 家庭列表 | 极低 | 300-600秒 | 家庭很少变化 |
| 智能列表 | 低 | 300秒 | 智能偶尔变化 |

**禁用缓存**：

```toml
[cache]
device_list_ttl = 0
device_state_ttl = 0
```

### 缓存目录配置

```python
from pathlib import Path
from mijiaAPI_V2 import create_api_client

# 自定义缓存目录
api = create_api_client(
    credential=credential,
    cache_dir=Path("/tmp/mijia_cache")
)
```

```python
redis_client = redis.Redis(
    host='localhost',
    port=6379,
    db=0,
    password='your_password',
    decode_responses=True,
    max_connections=50,        # 最大连接数
    socket_timeout=5,          # 套接字超时
    socket_connect_timeout=5,  # 连接超时

    credential=credential,
    redis_client=redis_client
)
```

### Redis集群配置

```python
from redis.cluster import RedisCluster

# Redis集群
redis_client = RedisCluster(
    startup_nodes=[
        {"host": "redis1.example.com", "port": 6379},
        {"host": "redis2.example.com", "port": 6379},
        {"host": "redis3.example.com", "port": 6379},
    ],
    decode_responses=True,
    password='your_password'
)

    credential=credential,
    redis_client=redis_client
)
```

### Redis Sentinel配置

```python
from redis.sentinel import Sentinel

# Redis Sentinel
sentinel = Sentinel([
    ('sentinel1.example.com', 26379),
    ('sentinel2.example.com', 26379),
    ('sentinel3.example.com', 26379),
], socket_timeout=0.1)

# 获取主节点
redis_client = sentinel.master_for(
    'mymaster',
    socket_timeout=0.1,
    password='your_password',
    decode_responses=True
)

api = create_api_client(
    credential=credential,
    redis_client=redis_client
)
```

## 日志配置详解

### 日志级别

```toml
[logging]
level = "INFO"
```

**级别说明**：

| 级别 | 用途 | 输出内容 |
|-----|------|---------|
| DEBUG | 开发调试 | 所有日志，包括详细的请求/响应 |
| INFO | 一般信息 | 关键操作、性能指标 |
| WARNING | 警告信息 | 潜在问题、降级操作 |
| ERROR | 错误信息 | 错误和异常 |
| CRITICAL | 严重错误 | 系统级错误 |

### 日志格式

**JSON格式**（推荐用于生产环境）：

```toml
[logging]
format = "json"
```

输出示例：

```json
{
  "timestamp": "2026-03-13T12:00:00.123Z",
  "level": "INFO",
  "logger": "mijiaAPI_V2.services.device_service",
  "message": "获取设备列表成功",
  "user_id": "123456",
  "home_id": "789",
  "device_count": 15,
  "duration_ms": 150,
  "cache_hit": true
}
```

**文本格式**（推荐用于开发环境）：

```toml
[logging]
format = "text"
```

输出示例：

```
2026-03-13 12:00:00.123 INFO [device_service] 获取设备列表成功 user_id=123456 home_id=789 device_count=15 duration_ms=150 cache_hit=true
```

### 日志文件轮转

```toml
[logging]
file = "logs/mijia.log"
max_bytes = 10485760      # 10MB
backup_count = 5          # 保留5个备份
```

**文件命名**：
- `mijia.log` - 当前日志
- `mijia.log.1` - 第1个备份
- `mijia.log.2` - 第2个备份
- ...
- `mijia.log.5` - 第5个备份

### 多日志输出

```python

# 获取logger
```

## 凭据配置

### 凭据存储路径

凭据文件用于保存用户的认证信息，支持灵活的路径配置：

```toml
[security]
# 凭据存储路径
credential_path = ".mijia/credential.json"
```

**支持的路径格式**：

1. **相对路径**（推荐用于项目级配置）
   ```toml
   credential_path = ".mijia/credential.json"
   ```
   - 相对于项目根目录（包含 `pyproject.toml` 的目录）
   - 适合将凭据保存在项目目录中
   - 便于项目隔离和版本控制（记得添加到 `.gitignore`）

2. **用户目录路径**（推荐用于全局配置）
   ```toml
   credential_path = "~/.mijia/credential.json"
   ```
   - 使用 `~` 开头会自动展开为用户主目录
   - 适合多个项目共享同一凭据
   - 凭据保存在用户目录，更安全

3. **绝对路径**
   ```toml
   credential_path = "/var/lib/mijia/credential.json"
   ```
   - 使用完整的绝对路径
   - 适合服务器部署场景

**路径解析规则**：

```python
from pathlib import Path

# 相对路径 -> 相对于项目根目录
".mijia/credential.json" 
# 解析为: /path/to/project/.mijia/credential.json

# 用户目录 -> 展开 ~
"~/.mijia/credential.json"
# 解析为: /Users/username/.mijia/credential.json

# 绝对路径 -> 直接使用
"/tmp/credential.json"
# 解析为: /tmp/credential.json
```

### 凭据文件安全

凭据文件会自动设置为仅所有者可读写（权限 `0o600`）：

```toml
[security]
credential_file_mode = 384  # 八进制 0o600 = 十进制 384
```

**权限说明**：
- `0o600` = 仅所有者可读写
- 其他用户无法读取凭据文件
- 自动应用于新创建的凭据文件

### 使用示例

**示例1：使用默认路径**

```python
from mijiaAPI_V2 import create_auth_service

# 使用配置文件中的默认路径
auth_service = create_auth_service()

# 登录并保存凭据
credential = auth_service.login_by_qrcode()
auth_service.save_credential(credential)  # 保存到默认路径

# 加载凭据
credential = auth_service.load_credential()  # 从默认路径加载
```

**示例2：使用自定义路径**

```python
from mijiaAPI_V2 import create_auth_service

auth_service = create_auth_service()

# 保存到自定义路径
credential = auth_service.login_by_qrcode()
auth_service.save_credential(credential, "custom_path.json")

# 从自定义路径加载
credential = auth_service.load_credential("custom_path.json")
```

**示例3：多用户凭据管理**

```python
from mijiaAPI_V2.infrastructure.credential_store import FileCredentialStore
from pathlib import Path

# 为不同用户使用不同的凭据文件
store_a = FileCredentialStore(default_path=Path(".mijia/user_a.json"))
store_b = FileCredentialStore(default_path=Path(".mijia/user_b.json"))

# 保存用户A的凭据
store_a.save(credential_a)

# 保存用户B的凭据
store_b.save(credential_b)

# 加载凭据
credential_a = store_a.load()
credential_b = store_b.load()
```

### 凭据自动刷新

凭据过期时可以自动刷新：

```python
from mijiaAPI_V2 import create_auth_service

auth_service = create_auth_service()

# 加载凭据
credential = auth_service.load_credential()

# 检查并刷新
if credential and credential.is_expired():
    print("凭据已过期，正在刷新...")
    credential = auth_service.refresh_credential(credential)
    auth_service.save_credential(credential)
    print("凭据刷新成功")
```

### 凭据迁移

如果需要迁移凭据到新路径：

```python
from mijiaAPI_V2.infrastructure.credential_store import FileCredentialStore
from pathlib import Path

# 从旧路径加载
old_store = FileCredentialStore(default_path=Path("~/.mijia/credential.json"))
credential = old_store.load()

# 保存到新路径
new_store = FileCredentialStore(default_path=Path(".mijia/credential.json"))
new_store.save(credential)

print("凭据迁移完成")
```

### 3. 配置验证

```python
def validate_config(config):
    """验证配置"""
    # 检查必需配置
    assert config.get("API_BASE_URL"), "缺少API_BASE_URL"
    
    # 检查Redis配置
    if config.get_bool("REDIS_ENABLED"):
        assert config.get("REDIS_HOST"), "启用Redis但未配置REDIS_HOST"
    
    # 检查超时配置
    timeout = config.get_int("NETWORK_DEFAULT_TIMEOUT")
    assert 1 <= timeout <= 300, f"超时时间 {timeout} 不在有效范围"

# 使用
config = ConfigManager()
validate_config(config)
```

## 故障排查

### 问题1：配置不生效

**检查优先级**：

```python
from mijiaAPI_V2.core.config import ConfigManager

config = ConfigManager()

# 检查配置来源
print(f"API_BASE_URL: {config.get('API_BASE_URL')}")
print(f"来源: {config.get_source('API_BASE_URL')}")
# 输出: 来源: environment（环境变量）/ file（配置文件）/ default（默认值）
```

### 问题2：Redis连接失败

**检查连接**：

```python
import redis

try:
    client = redis.Redis(host='localhost', port=6379)
    client.ping()
    print("Redis连接成功")
except redis.ConnectionError as e:
    print(f"Redis连接失败: {e}")
```

### 问题3：日志文件权限错误

**检查权限**：

```bash
# 创建日志目录
mkdir -p logs

# 设置权限
chmod 755 logs
```
